# 登录准入

当前项目支持网页自助注册账号、账号密码登录和登录用户修改自己的账号配置。登录只做访问准入，不把画布、我的素材、生成记录或 AI API Key 保存到项目服务器。

## 配置文件

登录运行参数位于根目录：

```text
config/app.yaml
```

这里仅保存应用名、是否启用登录、是否开放注册、Cookie、会话时长、账号数据文件路径等运行参数。项目不通过环境变量或命令行参数传递这些配置。

账号数据不写入 YAML。用户通过网页注册或修改账号后，会写入：

```text
data/auth-users.json
```

`data/` 是运行期数据目录，默认不进入 Git。Docker Compose 已把宿主机 `./data` 挂载到容器 `/app/data`，用于保留注册账号。

## 注册和登录

首次访问应用时打开 `/login`：

- 已有账号可直接登录；
- 新用户可切换到“注册账号”，填写用户名、显示名称和密码；
- 注册成功后会自动登录并进入应用。

用户名规则：3-32 位小写字母、数字、下划线或短横线。密码长度：6-128 位。

## 修改自己的账号配置

登录后点击右上角用户名或“账号配置”图标，可以修改：

- 用户名；
- 显示名称；
- 密码。

修改密码时需要填写当前密码。修改用户名或显示名称后，会刷新当前登录会话。

## 数据保存边界

登录账号只用于访问准入。登录后，每个人的创作数据仍然保存在当前浏览器本地：

- 画布项目；
- 我的素材；
- 生图、视频生成记录；
- AI API Key 和模型渠道配置。

换电脑、换浏览器、清理浏览器数据后，这些本地数据不会自动跟随账号恢复。需要跨设备时，仍然使用配置弹窗里的 WebDAV 同步。

## 准入范围

启用登录后：

- 普通页面会跳转到 `/login`；
- 未登录访问 Next.js API route 会返回 `401`；
- `/webdav-proxy` 也需要登录后才能调用；
- `/api/auth/register`、`/api/auth/login`、`/api/auth/logout`、`/api/auth/session` 是登录相关公开接口。
